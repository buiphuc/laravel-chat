# API Endpoints — phucbui/laravel-chat

> Complete API reference for all endpoints. Routes are dynamically generated for each actor type; prefix depends on config.

## Route Registration

Routes are automatically generated by `ChatServiceProvider` based on `config('chat.actors')`. Default example:

| Actor | Prefix | Middleware Stack |
|---|---|---|
| `super_admin` | `api/super-admin/chat` | `auth:sanctum`, `chat.resolve_actor:super_admin` |
| `admin` | `api/admin/chat` | `auth:sanctum`, `chat.resolve_actor:admin` |
| `client` | `api/chat` | `auth:sanctum`, `chat.resolve_actor:client` |

## Middleware

| Alias | Class | Description |
|---|---|---|
| `chat.resolve_actor` | `ResolveActorMiddleware` | Resolves current user as an actor, attaches `chat_actor` + `chat_actor_name` to request |
| `chat.capability` | `CheckCapabilityMiddleware` | Checks if actor has a specific capability |
| `chat.room_access` | `ChatRoomAccessMiddleware` | Checks if actor can access the room (member or `can_see_all_rooms`) |

### POST `/{prefix}/rooms` — Create room

**Direct room** (1v1):
```json
{
  "target_id": 5,
  "target_type": "App\\Models\\User"
}
```

**Group room** (requires `can_create_group` capability):
```json
{
  "name": "Team Chat",
  "max_members": 20,
  "participant_ids": [2, 3, 5],
  "participant_type": "App\\Models\\User",
  "metadata": {"department": "sales"}
}
```

**Validation:**
| Field | Rules |
|---|---|
| `target_id` | `required_without:name`, `integer` |
| `target_type` | `required_with:target_id`, `string` |
| `name` | `required_without:target_id`, `string`, `max:255` |
| `max_members` | `nullable`, `integer`, `min:2` |
| `participant_ids` | `nullable`, `array` |
| `participant_type` | `required_with:participant_ids`, `string` |

**Auto-routing**: When `from_actor` (client) creates a direct room and auto-routing is enabled, the package automatically finds a suitable admin instead of using the provided `target_id`.

**Response** `201`: `RoomResource`

### PUT `/{prefix}/rooms/{room}` — Update room

**Middleware**: `chat.room_access`
**Capability**: `can_manage_participants`

```json
{
  "name": "New Room Name",
  "metadata": {"pinned": true}
}
```

**Response** `200`: `RoomResource`

### DELETE `/{prefix}/rooms/{room}` — Delete room (soft delete)

**Middleware**: `chat.room_access`
**Capability**: `can_manage_participants`

**Response** `200`:
```json
{"message": "Room deleted."}
```

### POST `/{prefix}/rooms/{room}/messages` — Send message

**Middleware**: `chat.room_access`

```json
{
  "body": "Hello, how can I help you?",
  "type": "text",
  "parent_id": null,
  "metadata": {"priority": "high"}
}
```

**File attachments supported** (multipart/form-data):
| Field | Rules |
|---|---|
| `body` | `required_without:attachments`, `string`, `max:5000` |
| `type` | `sometimes`, `in:text,image,file,system` |
| `parent_id` | `sometimes`, `integer` |
| `metadata` | `sometimes`, `array` |
| `attachments` | `sometimes`, `array` |
| `attachments.*` | `file` |

**Processing flow:**

```mermaid
sequenceDiagram
    participant Client as Host App
    participant MW as Middleware
    participant Ctrl as MessageController
    participant MsgSvc as MessageService
    participant AttSvc as AttachmentService
    participant Repo as MessageRepository
    participant DB as Database
    participant DRV as SocketDriver
    participant EVT as MessageSent Event
    participant NTF as Notification

    Client->>MW: POST /rooms/{room}/messages
    MW->>MW: ResolveActor → ChatRoomAccess
    MW->>Ctrl: Validated request
    Ctrl->>MsgSvc: send(room, actor, MessageData)
    MsgSvc->>Repo: create(data)
    Repo->>DB: INSERT chat_messages
    MsgSvc->>DB: UPDATE chat_rooms.last_message_at
    MsgSvc->>DRV: broadcast(channel, 'message.sent', data)
    MsgSvc->>EVT: event(MessageSent)
    alt notifications.enabled
        MsgSvc->>NTF: notify offline participants
    end
    MsgSvc-->>Ctrl: ChatMessage
    alt has attachments
        Ctrl->>AttSvc: upload(message, file)
        AttSvc->>DB: INSERT chat_attachments
    end
    Ctrl-->>Client: 201 MessageResource
```

**Response** `201`: `MessageResource`

### POST `/{prefix}/rooms/{room}/read` — Mark as read

**Middleware**: `chat.room_access`

**Response** `200`:
```json
{"message": "Marked as read."}
```

Fires `MessageRead` event → broadcasts read receipt.

### POST `/{prefix}/rooms/{room}/typing` — Typing indicator

**Middleware**: `chat.room_access`

```json
{"is_typing": true}
```

Fires `UserTyping` event → broadcasts to room.

### GET `/{prefix}/messages/search` — Search messages

| Param | Type | Rules | Description |
|---|---|---|---|
| `keyword` | string | `required`, `min:2` | Search keyword |
| `room_id` | integer | `sometimes` | Filter by room |
| `from_date` | date | `sometimes` | From date |
| `to_date` | date | `sometimes` | To date |

**Response** `200`: Paginated `MessageResource`

**Middleware**: `chat.room_access`
**Capability**: `can_manage_participants`

```json
{
  "actor_id": 5,
  "actor_type": "App\\Models\\User",
```

**Middleware**: `chat.room_access`

### DELETE `/{prefix}/rooms/{room}/participants/{participant}` — Remove participant

**Middleware**: `chat.room_access`
**Capability**: `can_manage_participants`

**Response** `200`
